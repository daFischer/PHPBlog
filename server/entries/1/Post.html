Alright then, let's get some kind of roadmap in here.

What's done already

  • I can post blog entries (as HTML file) complete with their own JS and CSS files These 3 files (per post) are combined server side, the readers won't even notice.
  • The CSS file is loaded as a scoped style element, so it only affects whatever post it's loaded for.
  • It works without needing any JS, so those of you prefering to use NoScript won't have any problems ;) Some posts might be showing JavaScript, though. Since this is a blog mainly about WebDev that kinda can't be avoided.
  • All requests to .../blog/... are redirected to .../blog/index.php where the visible sites are created from several template HTML files. This allows for centralized Page Building, avoiding the dreaded Spaghetti code.
  • Right now there are only two sites, this one and another, hidden one.
